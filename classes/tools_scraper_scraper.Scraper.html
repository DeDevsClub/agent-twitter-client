getTweetV2

  • getTweetV2(id, options?): Promise<null | Tweet>

  • Fetches a single tweet by ID using the Twitter API v2. Allows specifying optional expansions and fields for more detailed data.

    Parameters

    • id: string

      The ID of the tweet to fetch.

    • Optional options: {
      expansions?: TTweetv2Expansion[];
      mediaFields?: TTweetv2MediaField[];
      placeFields?: TTweetv2PlaceField[];
      pollFields?: TTweetv2PollField[];
      tweetFields?: TTweetv2TweetField[];
      userFields?: TTweetv2UserField[];
      } = defaultOptions

      Optional parameters to customize the tweet data.

      • Optional expansions?: TTweetv2Expansion[]

        Array of expansions to include, e.g., 'attachments.poll_ids'.

      • Optional mediaFields?: TTweetv2MediaField[]

        Array of media fields to include, if the tweet includes media, e.g., 'url', 'preview_image_url'.

      • Optional placeFields?: TTweetv2PlaceField[]

        Array of place fields to include, if the tweet includes location data, e.g., 'full_name', 'country'.

      • Optional pollFields?: TTweetv2PollField[]

        Array of poll fields to include, if the tweet has a poll, e.g., 'options', 'end_datetime'.

      • Optional tweetFields?: TTweetv2TweetField[]

        Array of tweet fields to include, e.g., 'created_at', 'public_metrics'.

      • Optional userFields?: TTweetv2UserField[]

        Array of user fields to include, if user information is requested, e.g., 'username', 'verified'.

    Returns Promise<null | Tweet>

    • The tweet data, including requested expansions and fields.
    • Defined in src/tools/scraper/scraper.ts:641

getTweetWhere

  • getTweetWhere(tweets, query): Promise<null | Tweet>

  • Fetches the first tweet matching the given query.

    Example:

    const timeline = scraper.getTweets('user', 200);
    const retweet = await scraper.getTweetWhere(timeline, { isRetweet: true });

    Parameters

    • tweets: AsyncIterable<Tweet, any, any>

      The AsyncIterable of tweets to search through.

    • query: TweetQuery

      A query to test all tweets against. This may be either an object of key/value pairs or a predicate. If this query is an object, all key/value pairs must match a Tweet for it to be returned. If this query is a predicate, it must resolve to true for a Tweet to be returned.

      • All keys are optional.
      • If specified, the key must be implemented by that of Tweet.

    Returns Promise<null | Tweet>

    • Defined in src/tools/scraper/scraper.ts:570
